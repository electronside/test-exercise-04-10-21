=== GIT внесение изменений ===

Рабочее окружение
Вы закончили работу над задачей для клиента у вас есть рабочий код, который был создан без использования git. Клиент создал пустой репозиторий git@example.com:example/test.git и расшалил для вас доступ.

Задание
Опишите ваши дальнейшие действия что бы ваш код появился на ветке master в репозитории git@example.com:example/test.git


1. Создам рабочую папку проекта
	
	git init 

	git config добавит данные о том кто и когда это сделал
		   чтоб при командной работе было видно что и когда я сделал
		   в файле .git/config запишет эту информацию.

	git config --global user.name "User Name"
	git config --global user.email user@mail.com

	посмотреть конфиг

	можно убрать дублирование инфы о пользователе
	git config --remove section user

	git config --list --global

	или на прямую .git/config

	git status

	On branch master
	No commits yet
	Untracked files:
  	(use "git add <file>..." to include in what will be committed)
        	git.txt
        	linux_navigation.txt

		В индексе пока что файлы проекта untracked неотслеживаются

	git add .

	git status

	On branch master
	No commits yet
	Changes to be committed:
  	(use "git rm --cached <file>..." to unstage)
        new file:   git.txt
        new file:   linux_navigation.txt

		Теперь файлы проиндексированны

	git commit -m "Test version" вовремя закомитить
	чтоб откатить если накосячил

		или

	git commit -am "Test version"
		
		все изменения из рабочей папки через индекс сразу в репозиторий
		флаг -a игнорит файлы которых не отслеживает git

	[master (root-commit) 76fdfd8] Test exercise ready — хэш коммита
 	2 files changed, 133 insertions(+)
 	create mode 100644 git.txt
 	create mode 100644 linux_navigation.txt

		После коммита файлы переходят в репозиторий

	git status

	On branch master
	nothing to commit, working tree clean

		Состояние файлов Index и Repo идентичны

		Отправляем в удаленный репозиторий	

	git remote add origin https://github.com/example/test.git
	git pull --rebase origin master
	git push origin master

	или как в документации

	git init
	git add .
	git commit -a -m 'first commit'
	git remote add origin http://git@example.com:example/test.git
	git push origin master
